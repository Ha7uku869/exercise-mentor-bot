# Current Task

更新日: 2026-06-08

## 目的

運動メンターMVPを、単なる運動記録アプリから
**就活向けの Personal AI Agent（自然文を構造化し、個人文脈を記憶して根拠ある提案をするAI）**
へ拡張する。

短時間で完成品を作ることが目的ではない。
面接で深掘りされても、設計・実装・失敗・改善を**自分の言葉で説明できる状態**を作ることが目的。

対象企業: トヨタ / NRI / LINEヤフー / NTTドコモ

## 開発ルール

LLM（Claude / Codex）は**実装代行者ではなく、メンターとして使う**。

- いきなりコードを実装させない。まず目的・関係ファイル・変更理由を自分で説明する
- 大きな実装を一気に進めない。小さい差分に分ける
- 理解せずにコピペしない
- 各実装後に「面接30秒説明」と「深掘り質問3つ」を作る
- テストせずに「動くはず」で判断しない

## 実装順（優先タスク）

依存関係を考慮した実装順。**上から順に進める。**

1. ~~**会話ログ復元**~~ ✅ 完了 (2026-06-08)
   - 「再訪時に会話が消える」友人FBの解消
   - ログイン時に `GET /api/logs/{user_id}` で取得し復元、user_id でユーザー分離
   - 動作確認: 入力→AI返答→リロード／ログアウト跨ぎでも時系列順に復元、ユーザー分離OK
   - 残課題: 件数制限（LIMIT）未対応・現状は全件取得。本番は直近N件＋ページネーションが必要
2. **LLM抽出テスト・評価基盤** ← 次にここ
   - 20〜40件のテストケース、保存すべき／すべきでない発話を pytest で検証
3. 記憶カードUI
   - `context_notes` を category 別に一覧表示し、AIが何を覚えているか可視化
4. ユーザー文脈グラフ
   - ノード／エッジの最小モデルで「なぜその提案か」を説明する材料を作る
5. 軽量Multi-Agent化
   - Extractor / Retriever / Coach / Safety を関数・モジュール分割で責務分離
6. README・面接資料
   - 構成図、30秒説明、2分説明、想定問答、現状の限界と本番化に必要な改善
