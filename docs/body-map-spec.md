  # 人体図機能 仕様メモ（body-map-spec）

  ## 部位タクソノミー（canonical = 正式リスト）
  DB・LLMツール・フロント描画の3者が共有する。11部位で固定。



  | slug | 日本語 | 代表種目 |
  |---|---|---|
  | chest | 大胸筋 | ベンチプレス |
  | shoulders | 肩 | サイドレイズ |
  | back | 背中 | 懸垂 |
  | biceps | 上腕二頭筋 | アームカール |
  | triceps | 上腕三頭筋 | ディップス |
  | forearms | 前腕 | ハンマーカール|
  | abs | 腹筋 | クランチ |
  | glutes | 臀筋 | ヒップスラスト |
  | quads | 前もも | スクワット |
  | hamstrings | 裏もも | デッドリフト |
  | calves | ふくらはぎ | カーフレイズ |

  ## 設計判断
  1. 粒度は「筋肉痛で種目を変える判断ができる最小単位」=10個。細かすぎるとLLMタグがブレ・図の塗り分け
  が曖昧・グラフ母数が細切れ。粗すぎると動機を表現できない。
  2. 自前slug(11個)を正とし、描画ライブラリの筋肉名へはフロント側のマッピング表で変換する。→ DB/LLM
  enumを安定させ、ライブラリ差し替えに強くする。
